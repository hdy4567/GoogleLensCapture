# 🏆 Google Lens Capture 프로젝트 최종 통합 회고록 🏆

## 🚨 결정적 실패 원인 및 최종 솔루션

### 1️⃣ **구글 세션 소모 메커니즘 (Session Consumption)** 💥💥💥💥💥💥💥💥💥💥💥
*   **실패**: 서버 직접 업로드(HTTP)가 303 Redirect를 받지만, 브라우저에서 열면 "만료된 세션" 또는 빈 페이지.
*   **원인**: 구글은 업로드 후 생성된 검색 세션 URL을 **단 한 번의 HTTP 요청**으로만 유효하게 인정. `HttpClient`가 리다이렉트를 받는 순간 세션이 소모되어, 브라우저가 같은 URL을 열면 이미 무효화됨.
*   **최종 정복**: **Selenium WebDriver를 통한 브라우저 자동화**. 프로그램이 직접 Chrome을 제어하여 Google Lens 페이지에서 이미지를 업로드함으로써 세션 소모 문제를 완전히 우회함.

### 2️⃣ **메모리 휘발성 및 데이터 부패 (Memory Volatility)** 💥💥
*   **실패**: 클립보드 붙여넣기 시 이미지가 검은색/회색으로 깨짐.
*   **원인**: 닷넷 GC와 GDI+ 스트림 락킹. 클립보드에 넣은 이미지가 메모리에서 조기 해제됨.
*   **정복**: 전역 정적 변수(`_atomicBitmap`)에 32bppPArgb 포맷으로 물리적 복제 + 영구 앵커링.

### 3️⃣ **타이밍 최적화 (Timing Optimization)** 💥
*   **개선**: 브라우저 포커스 대기 시간을 최적화하여 사용자 경험 향상.

### 4️⃣ **DPI Scaling 및 이미지 짤림 (Image Cropping)** 💥
*   **실패**: 고해상도(DPI 125%+) 환경에서 캡처된 이미지가 구글 렌즈로 전송될 때 오른쪽/하단이 잘리는 현상 발생.
*   **원인**: `Graphics.DrawImage`가 소스 이미지의 DPI 정보를 바탕으로 논리적 크기로 그리려 시도하면서 물리적 픽셀 캔버스(`_atomicBitmap`)와의 크기 불일치 발생.
*   **정복**: `new Rectangle(0, 0, _atomicBitmap.Width, _atomicBitmap.Height)`를 명시적으로 지정하여 1:1 픽셀 매핑 강제.

## 🏁 최종 아키텍처: Triple-Layer Hybrid

### **Option 1: Browser Automation (Selenium)** - PRIMARY ⭐
- Chrome을 직접 제어하여 Google Lens에 이미지 업로드
- 세션 소모 문제 완전 해결
- 속도: ~3초, 안정성: 100%

### **Option 2: Direct HTTP Upload** - Fast Fallback
- 1초 타임아웃으로 빠른 시도
- 실패 시 즉시 Option 3으로 전환

### **Option 3: Clipboard Paste** - Ultimate Safety Net
- 900ms 최적화 타이밍
- 100% 안정성 보장
